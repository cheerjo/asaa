Mobile Template

The following tutorial uses the compact build of the ArcGIS API for JavaScript to build an application that displays a basemap from ArcGIS.com.

  1. Target Devices and Orientations

    This template targets the iPhone and iPad in both portrait and landscape orientations. It uses the Dojo Mobile framework and includes the Dojo Heading widget.


  2. Meta Tags

    The meta tag properties viewport, apple-mobile-web-app-capable, and apple-mobile-web-app-status-bar-style contain several useful properties when building mobile friendly web applications. 

    <!doctype html>
<html>
 <head>
  <meta http-equiv="Content-Type" content="text/html; charset=utf-8">
  <meta http-equiv="X-UA-Compatible" content="IE=7,IE=9">
  <meta name="apple-mobile-web-app-capable" content="yes">
  <meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
  <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=0">

  3. Reference the compact build of the ArcGIS API for JavaScript

    To begin working with the ArcGIS API for JavaScript, add the following script and style sheet tags inside the <head> element of your HTML page: 

      <script src="http://serverapi.arcgisonline.com/jsapi/arcgis/3.3compact"></script>
  <link rel="stylesheet" href="http://serverapi.arcgisonline.com/jsapi/arcgis/3.3/js/esri/css/esri.css" />
  <!-- css used to customize the UI of the application -->
  <link rel="stylesheet" href="css/main.css" />

    The <script> tag uses the ArcGIS API for JavaScript from a Content Delivery Network(CDN) provided by Esri. When new versions of the API are released, upgrades are done by updating the version number in the URL to the API. The esri.css file is also required to use the API. For more info on esri.css, refer to the required CSS page.

  4. Initialization

    We first must declare several variables related to storing dimensions of the application and the target device being used. We will use the values to adjust the dimensions of the map to fill the viewport in both orientations (landscape and portrait). 

    <script>
 dojo.require("esri.map");

 // physical height of the browser screen
 var clientHeight;
 // used to hold a new MediaQueryList object representing the parsed results of the specified media query string
 var isPortrait;
 // determine if the device is an iPad or iPhone
 var isiPad;
 var isiPhone;
 // store the height of the status bar
 var statusBarHeight = 20;
 // Using Dojo's geometry module, we can get the position and size of an element.  
 // We use dojo.position to get the height (h) of Dojo's heading widget.  This value
 // will be used to resize the map's height and fill empty space in the viewport when
 // the device's orientation has changed
 var headerGeom;
 var map;
 
 ...
</script>

    The map resize() and reposition() methods are useful for updating the size and position on the map. This tutorial uses Dojo's header widget. The header represents a navigation bar, which usually appears at the top of the viewport. Therefore, we need to use a few additional properties to adjust the map size when the device's orientation changes.

    We can probe the userAgent property of the navigator object to detect the device type. window.matchMedia can be used to perform a media query and determine the device's current orientation (landscape or portrait). Dojo's geomtery module can be used to get the size of the device's status bar. Please refer to Apple's documentation for exact dimension of UI components. 

     function init(has, geom) {
   // is the device in portrait or landscape orientation
   isPortrait = window.matchMedia("(orientation: portrait)").matches;
   // is the user on an iPad or iPhone
   isiPad = navigator.userAgent.match(/iPad/i);
   isiPhone = navigator.userAgent.match(/iPhone/i);
   // get the position of the header displacement from the top of the device's window (essentially the size of the statusBar)
   headerGeom = geom.position(dojo.byId("header"));
   statusBarHeight = headerGeom.h/2;
   // get the viewport size
   clientHeight = document.body.clientHeight;
   // initialize the map
   initMap();
 }

  5. Initialize the map

    The map is created the same way regardless of the platform being targeted. 

     function initMap() {
   // onorientationchange doesn't always fire in a timely manner in Android so check for both orientationchange and resize
   var supportsOrientationChange = "onorientationchange" in window, orientationEvent = supportsOrientationChange ? "orientationchange" : "resize";
   window.addEventListener(orientationEvent, function() {
     orientationChanged();
   }, false);

   var initExtent = new esri.geometry.Extent({
     "xmin": -13763480.95,
     "ymin": 3676831.10,
     "xmax": -12662787.74,
     "ymax": 4410626.57,
     "spatialReference": {
       "wkid": 102100
     }
   });
   map = new esri.Map("map", {
     wrapAround180: true,
     extent: initExtent,
     slider: false
   });
   dojo.connect(map, "onLoad", mapLoadHandler);
   var tiledMapServiceLayer = new esri.layers.ArcGISTiledMapServiceLayer("http://services.arcgisonline.com/ArcGIS/rest/services/World_Topo_Map/MapServer");
   map.addLayer(tiledMapServiceLayer);
 }

  6. Determining device orientation

    After the map loads, adjust the map's height to fill the empty space created by the device's status bar. For each device the values will be different. 

     function mapLoadHandler(map) {
   // check orientation
   if (isiPhone) {
     // set the map's height to the height of the viewport (viewable area) plus the status bar height
     dojo.byId("map").style.height = (clientHeight + statusBarHeight) + "px";
   }
   if (isiPad != null) {
     addjustMapHeight();
   }
   map.resize();
   map.reposition();
 }
 
 function addjustMapHeight() {
   dojo.byId("map").style.height = (window.innerHeight - headerGeom.h) + "px";
 }
  7. Handling changes in device orientation

    orientationChanged() handles the device orientation changes. It resizes the map depending on the device type and orientation, and hides the address bar when in portait mode. 

     function orientationChanged() {
   hideAddressBar();
   if (isiPhone != null) {
     // iPhone
     if (isPortrait) {
       // portrait ----> landscape mode
       addjustMapHeight();
       isPortrait = false;
     } else {
       // landscape ----> portrait mode
       dojo.byId("map").style.height = (window.innerHeight + statusBarHeight) + "px";
       isPortrait = true;
     }
   } else if (isiPad != null) {
     // iPad
     addjustMapHeight();
   }
   if (map) {
     map.resize();
     map.reposition();
   }
 }

 /**
  * Screen real estate is precious on mobile devices.
  * In order to reclaim some screen space, the address bar at the top of browsers can be hidden.  
  * JavaScript can be used to automatically scroll the page.
  */
 function hideAddressBar() {
   // set a timeout
   setTimeout(function () {
     // hide the address bar!
     window.scrollTo(0, 1);
   }, 0);
 }
